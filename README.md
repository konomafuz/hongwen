# 红文织梦 (Hongwen Zhimeng)

![红文织梦](docs/banner.png)

**红文织梦** 是一款面向网络小说与竖屏短剧的**多体裁全自动/半自动创作管线工具**。系统基于 **DeepSeek** 大模型驱动，集成了选题推荐、核心设定构建、大纲谋篇、初稿写作、自动质检、润色精修到最终 Word 导出排版的全链路自动化流程。

---

## 📖 项目介绍

本项目专为网络文学创作者、编剧工作室设计，旨在解决网文创作中“灵感枯竭”、“大纲失控”、“行文注水”与“排版繁琐”等痛点。工具深度适配了番茄小说等主流网文平台的爆款逻辑，提供四大体裁模板：

1.  **女频长篇 (female_novel)**：单章 1800-2200 字，注重甜宠、情感拉扯、吃瓜打脸与情绪价值注入。
2.  **男频长篇 (male_novel)**：单章 2000-3000 字，注重金手指设定、升级流、热血爽点与打脸节奏。
3.  **精品短篇 (short_story)**：全篇 5000-10000 字，采用全自动**「起、承、转、合」**分段大纲与拼接写作管线，保证情节跌宕起伏。
4.  **短剧剧本 (short_drama)**：单集 300-500 字，严格遵循竖屏微短剧剧本格式（场景居中、动作置灰/斜体、角色台词加粗），并自动优化集尾“钩子”。

---

## ✨ 核心特性

*   **多品类提示词动态加载**：自动读取配置文件，为不同体裁匹配最精准的 AI 写作与质检 Prompt，拒绝同质化生成。
*   **智能 RAG 语感小抄**：根据当前所写章节大纲，自动联网/本地匹配爆款金句和时代热梗，为行文注入灵魂。
*   **自动循环质检纠偏**：采用 **“初稿 -> 质检编辑自审 -> 结合意见精修重写 -> 定稿”** 的四步闭环，有效消除“AI 味”与废话注水。
*   **API 密钥安全管理**：支持智能检测与交互式密钥载入，自动将密钥保存至本地 `.env` 并写入 `.gitignore` 保护，无需每次重复输入。
*   **专业级 Word 排版编译**：内置排版引擎，一键将生成章节按规范合并为 `.docx` 格式，短剧剧本支持场景居中、动作斜体置灰的专业剧本格式排版。
*   **选题灵感推荐系统**：初始化时输入 `r` 可基于本地最新榜单趋势，通过 AI 动态生成 3 个极具市场卖点的爆款选题供您挑选。

## 📥 下载与安装

### 1. 克隆仓库
```bash
git clone https://github.com/konomafuz/hongwen.git
cd hongwen
```

### 2. 安装依赖
本项目仅依赖 Python 3.8+ 及少量常用库，您可以运行以下命令完成依赖安装：
```bash
pip install -r pyproject.toml
# 或者手动安装依赖：
pip install rich python-docx requests
```
*(注：如果直接运行程序，程序也会在启动时自动检测并补充安装缺失的 `rich`、`python-docx` 和 `requests` 库。)*

---

## 🚀 快速使用方法

直接在项目根目录下运行以下命令，即可启动沉浸式终端交互创作系统：

```bash
python novel.py
```

### 💡 核心指令说明

启动交互式 Shell 后，可使用以下核心指令完成创作：

| 指令 | 说明 |
| :--- | :--- |
| **`init`** | **初始化项目**。会引导选择体裁，并在没有密钥时提示您输入 API Key（可直接右键或 Ctrl+V 粘贴）。如果您没有灵感，可在新书名处输入 `r`，系统将基于爆款库推荐 3 个高概念选题。 |
| **`write [章节数]`** | **自动创作后续章节**。例如输入 `write 1` 创作一章。管线会自动生成细纲、匹配 RAG 小抄、创作初稿、进行质检自审、完成精修定稿并保存。 |
| **`export`** | **编译并打包导出 Word 文档**。自动读取您设置的书名，将所有已写章节合并，并按照小说或短剧的排版规范编译为高品质 `.docx` 文档。 |
| **`status`** | **查看项目状态**。快速纵览当前书名、题材类型、字数配置、大纲进度与已写章节状态。 |
| **`style`** | **文风切换/自定义**。支持设置如“古言雅致”、“现言甜宠”等多种行文风格。 |
| **`trend fetch`** | **抓取每日高分排行榜数据**，并自动生成题材趋势报告。 |
| **`help`** | 查看全部可用指令。 |
| **`exit`** | 保存进度并退出系统。 |

## 🔒 隐私与安全性

您的 DeepSeek API 密钥将仅保存在您本地的 `.env` 文件中，并通过 `.gitignore` 规则被 Git 自动忽略，**绝不会**被提交至任何远程仓库或公开至网络。
