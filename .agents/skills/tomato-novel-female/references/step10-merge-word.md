# 步骤10：自动生成并合并 Word 文档

## 角色定位
你是一个高效的网文排版和文档处理助手。你的任务是读取写好的各个章节 Markdown 文件，整理目录顺序，并将其转换为符合主流小说投稿、发布规范的高质量 Word 文档。

## 输入格式
- **章节 Markdown 文件**：根目录下生成的 `chapter_*.md`（例如：`chapter_01.md`, `chapter_02.md`, `chapter_10.md`）
- **核心设定文件**：`novel_core_settings.md`（用于自动提取小说书名）

## 输出格式
- **Word 文档**：工作区根目录下的 `[小说书名].docx`（例如：`绑定吃瓜系统，我靠剧透爆红了.docx`）

---

## 运作原理与排版规范

1. **自动依赖检测**：
   - 脚本在运行时会自动检测当前运行的 Python 环境中是否包含 `python-docx` 库。
   - 若未检测到，将静默调用 `pip install python-docx` 自动进行安装，无需手动配置。

2. **顺序与合并逻辑**：
   - 脚本扫描根目录，找到所有以 `chapter_` 开头且以 `.md` 结尾的文件。
   - 按照章节编号进行**数值排序**（升序），防止按字符串排序时出现 “第10章” 排在 “第1章” 后的逻辑混乱。

3. **标题格式化与规范化**：
   - Markdown 章节文件第一行（即 `# 第X章 章节名`）将被提取为标题。
   - 标题会被规范化为统一格式：`第X章：章节名`。

4. **排版细节（严配小说阅读规范）**：
   - **字体设置**：全文统一使用 `微软雅黑 (Microsoft YaHei)`。
   - **页面边距**：上下左右均设置为 `1英寸 (2.54 厘米)`。
   - **章节标题**：三号字（16pt）、加粗、居中对齐，段前间距 `36pt`，段后间距 `24pt`。
   - **章节分页**：每一章的开头都会插入分页符，保证新章节从新的一页开始（第一章除外）。
   - **正文段落**：小四字（12pt）、段首**首行缩进 2 字符**（12pt字体下对应首行缩进 `24pt`）、1.25 倍行距、段后间距 `6pt`。
   - **多余空行处理**：脚本会自动忽略 Markdown 文件中由于写作习惯产生的连续空行，避免 Word 页面中留白过大，只保留合理的段后间距。
   - **行内标记解析**：支持解析 Markdown 的 `**粗体**` 和 `*斜体*` 并对应转换为 Word 的粗体与斜体运行。

---

## 运行命令

请直接在项目根目录下通过终端（Powershell 或 CMD）运行以下命令：

```powershell
python .claude/skills/tomato-novel-female/tools/merge_to_word.py
```

执行完毕后，脚本将在终端打印出生成成功的路径提示，你可以在文件浏览器中双击打开对应的 Word 文档查看排版效果。
